# План регрессионного тестирования для Task Service (ЛР17)

**Статус:** Draft  
**Версия:** 1.0  
**Ответственный:** dasakovalenko334-12  
**Покрывает Issue:** #1  

## Цель
Гарантировать, что изменения в коде не сломали существующий функционал сервиса задач (CRUD, валидация, фильтрация, контракты API).

## Приоритеты
- **P1 (Critical):** Критический путь, без которого продукт не работает (создание, просмотр, удаление).
- **P2 (High):** Важные функции, влияющие на удобство и безопасность (фильтрация, валидация, статусы).
- **P3 (Medium):** Вспомогательные функции и негативные сценарии.

## Сценарии регрессии

### Критический путь (CRUD)
| ID | Сценарий | Приоритет | Ожидаемый результат | Риск при провале |
| :--- | :--- | :--- | :--- | :--- |
| REG-01 | Создание новой задачи (POST /tasks) | P1 | Задача создаётся, возвращается 201, ID задачи в теле ответа | Потеря возможности добавлять задачи |
| REG-02 | Получение списка задач (GET /tasks) | P1 | Возвращается JSON-массив, статус 200, сортировка по дате создания | Пользователи не видят список задач |
| REG-03 | Получение одной задачи по ID (GET /tasks/{id}) | P1 | Возвращается JSON задачи, статус 200 | Невозможность просмотра деталей задачи |
| REG-04 | Обновление задачи (PUT /tasks/{id}) | P1 | Задача обновляется, возвращается 200, поля изменены | Невозможность редактировать задачи |
| REG-05 | Удаление задачи (DELETE /tasks/{id}) | P1 | Задача удаляется, статус 204, при повторном запросе — 404 | Невозможность очистки списка задач |

## Покрытие требований
- **CRUD:** Покрыто сценариями REG-01 – REG-05.
- **Контракты данных:** Покрыто сценариями REG-06 – REG-09.
- **Фильтрация:** Покрыто сценариями REG-10 – REG-12.
- **Негативные сценарии:** Покрыто сценариями REG-13 – REG-15.

## Стратегия выполнения
1.  Выполнить все сценарии P1. Если хотя бы один падает — стоп, релиз запрещён.
2.  Выполнить сценарии P2. Если есть ошибки — зафиксировать, но релиз возможен с оговорками.
3.  Выполнить сценарии P3. Ошибки фиксируются как бэклог.

## Регрессионные сценарии

| ID | Сценарий | Приоритет | Описание |
| :--- | :--- | :--- | :--- |
| REG-01 | Создание задачи | P1 | Проверка, что API принимает POST-запрос с валидными данными и возвращает 201 Created. |
| REG-02 | Получение списка задач | P1 | Проверка, что GET /tasks возвращает список хотя бы с одной созданной задачей. |
| REG-03 | Получение задачи по ID | P1 | Проверка, что GET /tasks/{id} возвращает данные конкретной задачи. |
| REG-04 | Обновление статуса задачи | P2 | Проверка, что PATCH /tasks/{id} корректно меняет статус (например, на "done"). |
| REG-05 | Удаление задачи | P1 | Проверка, что DELETE /tasks/{id} удаляет задачу и последующие запросы возвращают 404. |
| REG-06 | Валидация пустого названия | P2 | Проверка, что создание задачи без названия возвращает ошибку 400. |
| REG-07 | Валидация слишком длинного названия | P2 | Проверка, что название длиннее N символов отклоняется. |
| REG-08 | Фильтрация по статусу | P2 | Проверка, что запрос с query-параметром ?status=done возвращает только задачи с этим статусом. |
| REG-09 | Фильтрация по исполнителю | P2 | Проверка фильтрации по полю assignee. |
| REG-10 | Обработка несуществующего ID | P1 | Проверка, что запрос к несуществующей задаче возвращает 404, а не 500. |
| REG-11 | Проверка идемпотентности удаления | P2 | Проверка, что повторный DELETE для уже удалённой задачи не ломает сервис (обычно 404). |
| REG-12 | Проверка формата даты | P2 | Проверка, что поле created_at возвращается в формате ISO 8601. |
| REG-13 | Проверка пагинации | P2 | Проверка, что при большом количестве задач работает limit/offset. |
| REG-14 | Проверка обработки спецсимволов | P3 | Проверка, что в названии можно сохранить спецсимволы (если это разрешено ТЗ). |
| REG-15 | Проверка обработки невалидного JSON | P2 | Проверка, что POST с битым JSON возвращает понятную ошибку 400. |

### Тест на создание задачи (REG-01)
def test_create_task(client):
    payload = {"title": "Новая задача", "status": "new"}
    response = client.post("/tasks", json=payload)
    
    assert response.status_code == 201
    data = response.json()
    assert data["title"] == "Новая задача"
    assert "id" in data


def test_create_task_empty_title(client):
    payload = {"title": "", "status": "new"}
    response = client.post("/tasks", json=payload)
    
    assert response.status_code == 400
    assert "error" in response.json()
