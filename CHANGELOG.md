# Журнал изменений (Changelog)

Все значимые изменения по проекту фиксируются в этом файле.

## [0.3.0] - 2026-06-25

### Добавлено
- Реализован механизм валидации пустого названия задачи (защита от падения).
- Внедрена атомарная генерация ID для предотвращения дубликатов.
- Добавлен базовый слой логирования критических ошибок при старте.

### Изменено
- Обновлена структура вывода списка задач (исправлен формат статуса «Done»).
- Переработана логика обработки исключений в модуле создания задач.

### Исправлено
- Устранено падение программы при вводе спецсимволов в описание задачи.
- Исправлена ошибка отображения статуса «Done» в списке задач.
- Исправлена логика генерации ID, приводившая к дубликатам при перезапуске.

### Известные ограничения
- Модуль хранения данных остаётся уязвимым: повреждённый JSON-файл по-прежнему приводит к аварийному завершению программы (Critical-дефект открыт).
- Отсутствует обработка таймаутов при записи файлов (риск зависания на медленных дисках).
- Поиск задач чувствителен к регистру (требуется доработка нормализации строк).

### Почему эта версия готова к выпуску
Выбрана версия 0.3.0, так как проект прошёл этап стабилизации базовых сценариев: закрыты дефекты валидации ввода и генерации ID, что делает основные команды CLI безопаснее. Несмотря на наличие Critical-дефекта в модуле хранения, версия считается значимым шагом вперёд по сравнению с предыдущими сборками. Выпуск оформлен для демонстрации навыков сопровождения и подготовки релизов.
