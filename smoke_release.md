# Дымовой набор проверок для релиза (Smoke Release Tests)

**Проект:** Task Service (Лабораторная работа 17)  
**Цель:** быстрая проверка базовой работоспособности сервиса задач перед релизом.  
**Статус:** готов к использованию.  
**Исполнитель:** dasakovalenko334-12

## Критерии прохождения (Pass/Fail Criteria)

Сборка считается **STABLE (стабильной)**, если успешно пройдены все 10 проверок (SMK-01 — SMK-10).  
Если хотя бы одна проверка завершается с ошибкой — сборка **REJECTED (отклонена)**.

---

## Чек-лист проверок

### SMK-01: Проверка запуска приложения
**Цель:** убедиться, что сервис стартует без ошибок.  
**Шаги:**
1. Перейти в корневую директорию проекта.
2. Запустить сервис командой: `uvicorn main:app --reload`.
3. Дождаться сообщения `Uvicorn running on http://127.0.0.1:8000`.
**Ожидаемый результат:** сервис запущен, в консоли нет ошибок импорта или синтаксических ошибок. Статус: **UP**.

### SMK-02: Проверка эндпоинта здоровья (Health Check)
**Цель:** подтвердить, что приложение отвечает на базовые HTTP-запросы.  
**Шаги:**
1. Выполнить запрос: `curl http://localhost:8000/health`.
**Ожидаемый результат:** ответ `200 OK` с JSON: `{"status":"healthy"}`.

### SMK-03: Проверка списка задач (GET /tasks)
**Цель:** убедиться, что API отдаёт список задач (даже пустой).  
**Шаги:**
1. Выполнить запрос: `curl http://localhost:8000/tasks`.
**Ожидаемый результат:** ответ `200 OK` с JSON-массивом: `[]`.

### SMK-04: Проверка создания задачи (POST /tasks)
